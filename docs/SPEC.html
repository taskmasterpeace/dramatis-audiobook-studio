1Purpose & positioning

One-liner: the first audiobook generator whose output sounds produced, not just narrated. Every competitor — including Castwright, the closest one — stops at voices. DRAMATIS treats a book the way film post-production treats a script: dialog is one stem of a mix, not the whole product.

2Goals & non-goals

Goals (v1.0)

  • Manuscript in (.txt / .md / .epub) → finished M4B out, overnight on an 8 GB consumer GPU.
  • Full cast: per-character designed or cloned voices, consistent across a series.
  • Cinematic layer: scene-derived ambience beds, text-anchored SFX, optional score.
  • Dual masters from one render: Immersive and Clean (voice-only, ACX-compliant).
  • Every stage drivable by humans (web UI) and agents (MCP) with feature parity.
  • English end-to-end first; architecture language-agnostic via calibration profiles (§11).

Non-goals (v1.0)

  • No cloud SaaS, accounts, or telemetry. Local-first; any cloud model use is explicit opt-in.
  • No PDF/MOBI ingest (deferred — messy extraction, low signal for effort).
  • No mobile apps; no voice marketplace; no CJK languages (deferred to post-1.0).
  • No real-time streaming synthesis (preview streaming is enough — §13 M3).

3Product principles

  1. The Script is the truth. All audio derives from the Production Script. Users and agents edit the script; audio follows. Never the reverse.
  2. Every claim cites the text. Analyzer output (speakers, scenes, cues) carries character-offset spans into the manuscript. Unattributable inference is flagged, never silently asserted.
  3. Deterministic by default. Same script + same seeds + same engine versions = same audio, bit-for-bit. Every render is reproducible and resumable.
  4. Surgical re-render. Editing one line re-synthesizes one line and re-mixes one scene. Cost of change is proportional to the change.
  5. Dialog is sacred. The sound design layer may never harm intelligibility. Clean master is always available; immersive mix is ducked against dialog by construction.
  6. Permissive parts only. Engines, weights, and libraries must be MIT / Apache / BSD / CC0 — or are optional plugins clearly labeled otherwise. The product is always shippable without the labeled ones.
  7. Agent-native. Anything a user can do, an agent can do through MCP. The UI is one client among several.

5The Production Script (IR)

The compiler target between prose and audio. Versioned (append-only revisions), diffable, JSON. Everything downstream renders from it.

{
  "book": "coalfall-01", "revision": 41,
  "entities": [
    { "id": "ent_7f2", "kind": "character",
      "names": ["Mara", "the smith's daughter"],   // aliases → one ID
      "voice": "vox_ab9", "series_ref": "coalfall/mara" }
  ],
  "scenes": [
    { "id": "scn_c4a", "chapter": 3,
      "span": [10422, 18845],                      // manuscript offsets
      "env": { "location": "forge, interior", "time": "night",
               "weather": "storm outside", "mood": "tense" },
      "ambience": { "spec": "coal forge interior, storm muffled...",
                    "gain_lu": -18, "seed": 771234 },
      "lines": [
        { "id": "lin_0392", "entity": "ent_7f2", "kind": "dialogue",
          "text": "You shouldn't have come back.",
          "emotion": { "anger": 0.6, "fear": 0.2 },
          "cite": [11031, 11067], "seed": 8842 }
      ],
      "cues": [
        { "id": "cue_11b", "at_line": "lin_0392", "at_word": 3,
          "spec": "heavy wooden door slams", "source": "library|generate",
          "cite": [11102, 11147], "gain_lu": -8 }
      ] } ]
}
  • Entities are IDs; names are labels. Mid-series renames, alias merges, and cross-book cast reuse are pointer edits — an entire class of cast-management features collapses into the data model.
  • Every scene/line/cue carries a cite span. The review UI shows the sentence that justified it; the analyzer cannot invent uncited structure (principle 2).
  • Manuscript is immutable. Re-import produces a new book version; the script re-compiles against it with a three-way diff for user-edited fields.

8Audio & mix specification

S1 · DIALOGNarration + character lines. The anchor. Dialog-normalized −18 LUFS integrated. Never processed by the immersive layer beyond mastering.
S2 · AMBIENCEOne loopable bed per scene, derived from scene.env. Sits 15–20 LU under dialog; sidechain duck −4 dB extra (attack 250 ms, release 1.2 s). Equal-power crossfade 2–4 s at scene bounds.
S3 · SFXOne-shots anchored to word timestamps (forced alignment, Qwen3-FA — transient at anchor-word onset +100 ms, ≥500 ms separation). Retrieval-first from the CC0 corpus (CLAP sim ≥ 0.25), procedural fallback, weak matches skipped and flagged. Per-cue confidence (word-match × alignment × source) in the QA report. Never within 150 ms ahead of a stressed word onset.
S4 · MUSICChapter cues/beds from book.json music: entries (text-anchored, prompt-specified). Sits ~20 LU under dialog; same ducking law as S2. muapi/Suno today (preview licensing); ACE-Step local planned. Engine + license recorded per cue in the QA report.

9Data model (SQLite)

series · books · chapters · entities · aliases · voices · scenes · lines · cues · renders · qa_events · calibration_profiles · jobs

  • series is the cast bible: series_ref → voice bindings outlive any one book (cross-book, even cross-series linking is a row, not a feature).
  • renders maps content hashes → blob paths + seeds + engine versions: the reproducibility ledger.
  • qa_events is append-only; acceptance decisions feed calibration (§11).
  • Voice artifacts stored as safetensors only — no pickle loading of shareable files, ever (a known RCE-class risk in the incumbent).

10GPU & model residency

  • Leases, not heuristics. A render task takes a lease on its model; the residency manager may only evict models with zero active leases. The evict-mid-render race is impossible by construction.
  • Budgeted admission. Tasks declare cost from the VRAM table (§7); admission blocks until budget fits. Free-VRAM floor of 1 GB maintained for the compositor/OS.
  • Degraded modes are first-class: CPU-only (Kokoro narration, no cloning), low-VRAM (sequential engine swapping), full (concurrent). Selected automatically at startup, overridable.
  • Batch packing per engine: width × max-tokens budget (wide for short dialogue, narrow for long prose) — the known-good approach for ~1.0 RTF full-book runs on 8 GB.

Calibration profiles: thresholds are per-(language, engine) rows seeded with defaults and tuned from the accept/reject history in qa_events — not constants baked per machine. Blocked lines auto-reroll once (new seed) before landing in the review queue, pinned to their cited manuscript span.

12Agent surface (MCP)

Thirteen workflow-level tools, stdio + HTTP, shipped in v0.1 and versioned with the command API:

ingest_manuscript · compile_script · get_script · patch_script (JSON-patch ops, the agent's editing surface) · list_entities · design_voice · assign_voice · link_series_character · render (book/chapter/scene/line scope) · qa_report · mix · export · job_status

Acceptance test: an agent can produce a finished book from a bare manuscript using only these tools — this is CI, not a demo.

15Legal hygiene (clean-room rules)

  • Never open the incumbent's source. Behavior may be specified from public docs, README, and observed use of the running product only. This spec is the boundary artifact.
  • All contributors affirm they have not copied FSL-licensed code; PR template includes the affirmation.
  • Dependency license allowlist enforced in CI: MIT / Apache-2.0 / BSD / CC0 / ISC. Copyleft and community licenses require an explicit, documented exception (currently one: §7¹).
  • Voice cloning requires recorded consent metadata stored with the voice artifact; cloned voices are non-exportable by default.
  • Project ships under Apache-2.0 with a NOTICE file. No CLA beyond DCO sign-off.
